# ParkVision 项目架构说明

## 总体架构

```text
车主端 H5 / 运营后台 / 数字孪生大屏
          |
          | REST API
          v
Spring Boot 后端业务服务
  - 订单状态机
  - 车位管理
  - AGV 调度队列
  - 计费规则
  - 告警追踪
          |
          | 结构化识别结果
          v
边缘 AI 推理服务
  - 车牌识别
  - 车位占用检测
  - 人员闯入检测
  - 车流预测
```

## 前端结构

前端采用 Vue3 + Vite 单页应用，主要页面包括：

- `DashboardView`：总览驾驶舱。
- `OwnerPortalView`：车主端 H5。
- `AdminConsoleView`：运营后台。
- `TwinView`：数字孪生沙盘。
- `AiVisionView`：AI 感知与预测演示。
- `DispatchCenterView`：调度中枢。
- `PricingEngineView`：动态定价。
- `SystemConfigView`：网关与配置。
- `IndoorMapView`：室内外接续导航。

## 后端结构

后端采用标准 MVC 分层结构：

```text
com.parkvision.cps
  controller/     REST API 接口层
  service/        业务逻辑层
  repository/     数据访问抽象与 fallback 实现
  domain/         领域对象与状态枚举
  dto/            前后端传输对象
  common/         统一响应、业务异常、全局异常处理
  config/         CORS 等应用配置
```

Service 层只依赖 `ParkVisionRepository` 接口。当前演示阶段由 `FallbackParkVisionRepository` 提供 fallback 数据源，后续可替换为 MySQL、Redis、RabbitMQ。

## 核心业务闭环

```text
AI 识别车牌
  -> 创建入场订单
  -> 分配车位
  -> 更新数字孪生库位
  -> 车流预测触发 Pre-Dispatch
  -> AGV 执行深浅库位迁移
  -> 车主取车与支付离场
```

## 后续增强方向

- 引入 MySQL，设计用户、车辆、订单、车位、告警、计费规则表。
- 引入 Redis，缓存车位状态和 AGV 坐标。
- 引入 RabbitMQ，处理调度任务异步执行。
- 引入 WebSocket，推送 AGV 坐标与告警。
- 引入 FastAPI，封装真实 AI 模型推理接口。
- 使用 Swagger/OpenAPI 自动生成接口文档。
