# ParkVision 项目架构说明

## 总体架构

```text
车主端 H5 / 运营后台 / 数字孪生大屏
          |
          | REST API / WebSocket
          v
Spring Boot 业务中台
  - 订单状态机
  - 车位管理
  - 调度队列
  - 计费规则
  - 告警追溯
          |
          | 结构化识别结果
          v
Python FastAPI AI 服务
  - 车牌识别
  - 车位占用检测
  - 人员闯入检测
  - 车流预测
```

## 前端架构

前端采用 Vue3 单页应用，页面拆成七个视图：

- `DashboardView`：总览驾驶舱。
- `OwnerPortalView`：车主端 H5。
- `AdminConsoleView`：运营后台。
- `TwinView`：数字孪生沙盘。
- `AiVisionView`：AI 感知演示。
- `DispatchCenterView`：调度中枢。
- `DemoGuideView`：答辩脚本。

前端通过 `src/api/parkvisionApi.js` 对接后端，后端不可用时 fallback 到 mock 数据，保证答辩演示稳定。

## 后端架构

后端采用 Spring Boot 分层结构：

- Controller：HTTP 接口。
- Service：业务逻辑。
- Model/Record：领域对象与 DTO。
- Infrastructure：内存数据仓库，后续可替换为 MySQL、Redis、RabbitMQ。

当前后端为了系统实现汇报，优先体现架构和接口闭环，业务严谨性后续再增强。

## 核心业务闭环

```text
AI 识别车牌
  -> 创建入场订单
  -> 分配车位
  -> 更新数字孪生库位
  -> 车流预测触发 Pre-Dispatch
  -> AGV 执行深浅库位迁移
  -> 车主取车与支付离场
```

## 可继续加分的方向

- 引入 MySQL，设计用户、车辆、订单、车位、告警、计费规则表。
- 引入 Redis，缓存车位状态和 AGV 坐标。
- 引入 RabbitMQ，处理调度任务异步执行。
- 引入 WebSocket，推送 AGV 坐标与告警。
- 引入 FastAPI，真实封装 AI 模型推理接口。
- 使用 Swagger/OpenAPI 自动生成接口文档。
