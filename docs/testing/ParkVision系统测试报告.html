ParkVision 智能停车系统测试报告

项目名称：ParkVision-System 智能停车 CPS 演示系统
测试主题：系统测试、功能性测试、静态测试、黑盒测试、白盒测试、非功能测试与可用性测试 测试日期：2026-05-18
报告修订日期：2026-05-19 测试环境：本地 Windows 环境，前端预览服务 http://localhost:5173，后端服务 http://localhost:8080
测试结论：本轮系统测试通过。功能测试 12/12 通过，黑盒异常测试 5/5 通过，性能测试所有场景成功率 100%。

1. 测试目标

本轮测试目标是证明 ParkVision 已经具备完整系统级演示能力，而不是停留在前端静态页面或单点模拟数据。测试重点如下：

编号 测试目标 验证方式 结论
T-01 验证真实数据链路 前端页面调用后端 API，后端读取 H2 文件数据库 通过
T-02 验证业务闭环 入场、订单、车位、调度、取车、支付、计费、导航、报表联动 通过
T-03 验证设备模拟链路 摄像头、闸机、充电桩、AGV 由后端统一模拟状态 通过
T-04 验证异常处理能力 非法订单号、缺少参数、急停恢复等黑盒用例 通过
T-05 验证服务层分支 订单状态变更、急停控制、异常分支的白盒覆盖 通过
T-06 验证非功能表现 接口并发压测、可启动性、持久化、可观测性 通过
T-07 形成汇报证据 生成截图、流程图、性能图、Markdown 报告和 PDF 通过

2.2 本轮范围边界

内容 当前处理方式 说明
真实物理摄像头、闸机、PLC、AGV 后端模拟真实协议风格数据 项目没有实体硬件，本轮用 ONVIF、Modbus/TCP、OCPP 风格字段模拟设备接入

登录鉴权 暂未接入 当前安全测试聚焦输入校验、业务异常和急停安全联动；生产化需补角色权限和审计

14. 测试结论

ParkVision 当前已经完成系统测试闭环。前端页面读取后端真实业务数据，订单、车位、调度、设备、计费、导航和管理台之间可以联动；数据库默认使用 H2 文件数据库持久化，仓库拉取后可直接启动并自动初始化；设备侧通过后端模拟摄像头、闸机、充电桩和 AGV 的真实协议风格字段参与演示。
