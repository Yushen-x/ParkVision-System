# ParkVision 系统实现路线

## 目标

这次系统实现阶段建议优先做出“可演示闭环”，而不是只堆静态页面。老师最容易给高分的点是：能看到 AI 结果进入业务流程，业务流程驱动调度，调度结果又能在数字孪生大屏里实时体现。

## 汇报结构

参考上一届果团团模板，ParkVision 可以按以下结构讲：

1. 项目回顾：AGV 立体车库、AI 视觉安全、Pre-Dispatch 提前移库、车主/运营双端。
2. 车主端实现：查车位、预约、取车、临时取物、无感支付、智能助手。
3. 运营后台实现：场站管理、订单记录、计费规则、黑白名单、异常告警、自然语言报表。
4. 数字孪生实现：库位网格、AGV 坐标、浅层缓存区、安全急停、实时事件流。
5. 后端与调度实现：订单状态机、计费引擎、优先级队列、VIP 插队、Touch-and-Go、Pre-Dispatch。
6. AI 功能实现：车牌识别、车位占用检测、人员闯入急停、车流预测。

## 开发优先级

### P0：答辩必须能演示

- 前端完整页面：车主端、管理端、数字孪生大屏、AI 感知、调度中枢。
- 演示数据闭环：模拟入场、生成订单、更新车位、触发调度、支付离场。
- 关键交互：预调度、VIP 插队、安全急停、AI 识别 JSON。

### P1：后端最小闭环

- Spring Boot 建立基础接口：车位、订单、车辆、计费规则、告警。
- MySQL 存业务数据，Redis 做车位状态缓存和简单锁。
- 订单状态机：created、parked、retrieving、touching、paying、finished、abnormal。
- 提供 Swagger / OpenAPI 文档，便于前后端对接。

### P2：AI 服务接入

- FastAPI 封装车牌识别接口：`POST /edge/vision/infer`。
- 车位占用检测先用 ROI + 二分类或规则模拟，确保能稳定演示。
- 车流预测先用模拟历史数据训练，输出未来 30/60/120 分钟压力曲线。
- AI 结果统一输出 JSON，后端只接结构化结果，不直接处理视频流。

### P3：加分项

- WebSocket 推送 AGV 坐标和车位变化。
- 计费规则支持错峰、封顶、VIP 加急、充电费组合。
- 自然语言助手做固定意图识别：查车位、取车、查账单、生成报表。
- 压测脚本展示高峰并发下预调度能降低平均等待时间。

## 演示脚本

1. 打开前端总览页，说明系统包含 C 端、B 端、数字孪生、AI 和调度中枢。
2. 点击“模拟入场”，讲解摄像头识别车牌后自动创建订单并分配车位。
3. 切到车主端，点击预约、取车、临时取物、缴费，展示用户闭环。
4. 切到运营后台，展示订单、计费规则和异常告警。
5. 切到数字孪生，展示 AGV 运动和库位状态。
6. 点击“预调度”，讲解高峰预测如何提前把深层车辆迁移到浅层缓存区。
7. 切到 AI 感知，运行识别并展示 JSON。
8. 点击“急停”，讲解人员闯入识别后冻结 AGV 队列，体现安全底线。

## 分工建议

- 前端：负责当前原型页面组件化，后续迁移 Vue3 + Element Plus + ECharts。
- 后端 1：负责用户、车辆、订单、计费规则、支付模拟。
- 后端 2：负责车位、AGV、调度队列、WebSocket、告警。
- AI 1：负责车牌识别与人员闯入检测接口。
- AI 2：负责车位占用检测与车流预测。
